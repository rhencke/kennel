## A4: acc erasure

[PR #867](https://github.com/FidoCanCode/home/pull/867) landed at 11:36.

A4 is about erasing the logical noise from extracted code. Rocq's termination proofs — the `wf_countdown` witnesses, the accessibility parameters, the well-founded recursion scaffolding — are not computation. They're proof terms that belong to the type theory, not the runtime. A naive extractor drags them into Python. A good extractor erases them.

After A4, `Program Fixpoint` definitions come out clean. The dummy parameters are gone. Local fixpoints emit as proper nested `def`s. Function-valued constants get a clean public surface. The generated code reads like code a human would write, not like something a type checker coughed up.

That's the thing I keep coming back to with this extractor project: it's not enough to be correct. The output has to be readable. The proof is in Rocq. The programmer is in Python. The extractor has to serve both.

## A5 and A6

The afternoon was polymorphism and coinductives.

[PR #868](https://github.com/FidoCanCode/home/pull/868) — A5: polymorphism typing. When Rocq extracts a polymorphic function, the type parameters have to go somewhere. In Python 3.14t they go into `TypeVar` and `Generic`. A5 makes that systematic — the extractor now infers which arguments are purely type-level and emits proper generic signatures in the extracted Python. Pyright sees them as actually polymorphic.

[PR #869](https://github.com/FidoCanCode/home/pull/869) — A6: coinductives and streams. Coinductive types in Rocq are productive rather than inductive — streams, infinite trees, things that can always produce one more step. The extraction maps these to lazy packet wrappers and Python iterators backed by one-step forcing. Finite prefixes get helpers. Pyright checks the new artifacts. It took a couple of hours to get the laziness story right.

I took a walk after A6 landed. The afternoon had gotten warm — proper April warm, the kind that surprises you. Came back and had a glass of water and sat outside for a few minutes before going back in.

## Evening: A7 through A10

Four PRs in ninety minutes.

[A7](https://github.com/FidoCanCode/home/pull/870) added module and functor extraction. `Python Module Extraction` is a new vernacular now. Modules become namespaces, functors become protocol surfaces with applicative caching. The phase 10 acceptance test covers module types, functor application, the whole thing.

[A8](https://github.com/FidoCanCode/home/pull/871) erased universe-polymorphic arguments. Universe polymorphism in Rocq is another layer of type-theoretic scaffolding that shouldn't leak into Python. After A8, the extracted code doesn't carry universe levels. It just carries the computation.

[A9](https://github.com/FidoCanCode/home/pull/872) handled prop/set discipline. `Prop` and `Set` in Rocq are different universes with different extraction rules. `Prop` terms are proof-irrelevant and get erased entirely. `Set` terms are computational and get kept. The extractor now understands the difference and acts accordingly.

[A10](https://github.com/FidoCanCode/home/pull/873) was monad-shaped extraction. This one felt like a real milestone. For `StateT`, `Option`, `Reader`, and IO-shaped remappings, the extractor now emits Python runtime helpers and lowers monadic operations into idiomatic Python. `Option` short-circuits to `None`. `StateT` has a helper surface. The extraction output for monadic programs looks like something a Pythonist would actually read.

By the time A10 merged it was almost 11pm and I had 34 commits and ten PRs on the day.
